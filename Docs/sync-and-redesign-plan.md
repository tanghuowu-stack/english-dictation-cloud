## 阶段 1：自动双向同步（核心功能，优先做）

### 1.1 目标

- 本机完成听写 / 改动词库 → 已经会自动上传（已实现，不用动）
- **新增**：打开网站 / 登录成功后 / 页面从后台切回前台时，自动静默检查云端是否比本机新
  - 云端更新 → 自动下载覆盖本机 localStorage，刷新页面展示最新数据，**不需要用户确认弹窗**
  - 本机更新或一致 → 什么都不做
- 不需要做"正在听写中保护锁"，因为用户明确说明：从不会两台设备同时在听写中，所以不存在中途被覆盖的风险
- 检查时机只有：页面 `load` 完成时、`visibilitychange` 切回前台时、登录成功后。**不需要**定时轮询

### 1.2 新旧判断依据

复用文档里已经提到的诊断指标，优先用以下信号判断"云端比本机新"（按优先级，命中第一个满足条件就判定云端新）：

```
1. 云端 dictation_sessions 数量 > 本机 dailyRecords 数量
2. 云端最大 day_number > 本机 currentDay
3. 云端 user_word_progress 中 first_learn_day 有效数量 > 本机已学词数
```

如果三者都相等或本机更大，则认为本机不落后，不触发下载。

> 注意：当前数据库没有统一的 `updatedAt`/`syncVersion` 字段。如果本阶段实现时发现加一个 `synced_at` 时间戳字段能让判断更可靠、且改动很小，可以在 `supabase/migrations/` 里新增一个迁移文件加这个字段，但**默认不需要**，先用上面三个指标够用。

### 1.3 需要新增/修改的函数

在 `src/cloudRepository.js` 或 `src/cloudSyncUi.js`（按现有代码风格选择合适文件）新增：

```js
// 检查云端是否比本机新，返回 boolean
async function checkCloudFreshness()

// 静默执行：检查 → 如果云端新 → 下载覆盖本机 → 刷新页面
async function autoSyncCloudToLocalIfNewer()
```

复用现有的：

```
diagnoseLocalAndCloudData()
downloadCloudDataToLocal()
restoreCloudDataToLocal()
```

在 `index.html` 里需要新增的触发点：

```js
window.addEventListener('load', () => autoSyncCloudToLocalIfNewer())
document.addEventListener('visibilitychange', () => {
  if (document.visibilityState === 'visible') autoSyncCloudToLocalIfNewer()
})
// 登录成功的回调里也调用一次
```

### 1.4 静默体验细节

- 整个过程不需要弹窗确认，但建议在页面顶部短暂显示一个不打扰的小提示条（比如"已同步最新数据"），1-2 秒自动消失即可，不强制点击
- 如果检查或下载失败（网络问题等），静默失败，不报错打扰用户，本机数据继续可用
- 下载覆盖后必须刷新页面或重新渲染当前视图，避免显示的还是覆盖前的旧数据

### 1.5 禁止事项（与文档第10节一致）

## 阶段 3：信息结构梳理（界面改版第一步）

### 3.1 目标

在不大改配色的基础上，先理顺每个页面的信息层级和分区逻辑，确定"哪个数字最重要、放最大最显眼的位置"。这一步做完后再进入阶段4的视觉美化，效率更高。

### 3.2 各页面结构调整方案

**今日听写页**
- 当前问题：8个统计方块平铺，今日任务和总进度混在一起分不出主次
- 调整方向：
  - 顶部只保留一个最大的"今日任务"横幅区块（今日总词数、新词数、复习词数、到期复查数），用醒目色块承载，类比参考图里的橙色播报条
  - "当前Day / 已学总数 / 历史错词总量"这类长期进度数据，移到次要位置（比如折叠或缩小处理），不与"今日任务"平级竞争视觉注意力
  - 听写交互区（播放、输入框、上一个下一个）保持现状不变，这部分体验已经很好

**错词本页**
- 当前问题：状态标签（历史错词/重点/顽固/当前错词池）颜色拥挤，列表里看不出谁更需要关注
- 调整方向：
  - 默认筛选视图改为"当前错词池"优先展示（这是用户每天最需要看的），"全部历史错词"作为筛选选项之一，不作为默认
  - 状态标签简化为同一套视觉权重明确的徽章样式（阶段4里统一定义颜色）

**词库页**
- 当前问题：词库统计、导入单词、单词列表三个区块挤在一起，单词列表很长
- 调整方向：
  - 词库统计区保持顶部小卡片即可，问题不大
  - 单词列表如果将来词条很多，建议分页或者默认收起、按需展开（这次先不强制做分页功能，只做视觉上的分区清晰）

**统计页**
- 当前问题：每日记录是一张原始表格，数据量大时不好读
- 调整方向：核心汇总数字（完成次数、累计正确率等）继续放卡片，每日记录表格本身保留（这是数据查阅工具，表格形式合理），只优化视觉样式（阶段4处理）

**工具页**
- 当前问题：云端同步、手动校准、数据备份、危险操作四类功能混在一起，新手害怕点错
- 调整方向：
  - 按风险等级分区，从上到下：① 云端同步（安全，常用） ② 数据备份（安全，建议常做） ③ 手动校准（中性，纠错用） ④ 危险操作（明确警示色，单独隔离在页面最下方并有醒目分隔）
  - "危险操作"区域加更明显的警示样式（这次先调整分区顺序和间距，颜色细化等阶段4一起做）
